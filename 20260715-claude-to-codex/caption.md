Claude封我2个Max号，我转Codex了

Claude Code 把我两个 Max 号都封了。

不是免费号，是两个付费的 Max 号。

第一个 Max 号没的时候，我还想着先申诉，手上的活用另一个顶着。结果第二个也没了。

这下真把我整不会了。

我用 Claude Code 不是聊着玩。平时写代码、查 Bug、改文章、跑 Skill，很多活都在里面。账号一挂，丢的不是一个聊天窗口，是整套工作突然停了。

所以这两天，我开始往 Codex 搬。

不是因为我突然觉得 Codex 天下第一，也不是看了哪个跑分。理由很现实：Claude 这边现在没法让我安稳干活，那我就先换一个能干活的。

真搬起来，倒没有想象中麻烦。

项目里的 CLAUDE.md 改成 AGENTS.md，原来的 Skills 继续带着，终端换成 Codex CLI。常用脚本本来就在仓库里，也不用重写。

当然，不是复制一下文件名就完事。权限、外部工具、Plugins / Apps 还得重新配，账号里的历史对话也搬不过来。但项目规则和代码都在 Git 里，至少不用从零再磨一遍。

Codex 这边我最想试的是多 Agent。我准备让主 Agent 先拆任务，再分出几个 Agent：一个读仓库、查依赖，一个在单独的 worktree 里改代码，另一个跑测试、看 diff。最后再把结果交回主任务收口。

并行不是一窝蜂开五个窗口。任务边界和文件范围不讲清楚，Agent 一样会互相踩。这个我会先从两三个独立任务试，不会一上来全自动。

当然，我也没觉得从此高枕无忧。OpenAI 一样有地区限制，Codex 不是保险箱。

两个 Max 号没了以后，我决定改一件事：不再把自己的工作方式锁在某个平台账号里。

以后规则、Skills、脚本都进 Git，关键改动及时提交。主力工具之外，再留个能随时接手的。账号没了就换工具，别让活跟着一起没。

Claude Code 先放一边。

Codex，我来了。
