# OpenClaw 终于接入企业微信了，保姆级配置教程来了

> 企业微信官方帮助中心已经上线 OpenClaw 接入说明。对想把 AI 真正接进团队工作流的人来说，这不是一次普通更新，而是一个明确信号：OpenClaw 已经可以正式进入企业微信场景了。

很多人前面其实只差最后一步。

OpenClaw 已经装好了，模型也能跑，终端里也能正常对话，但一到“怎么接进企业微信”就开始卡住：

- 机器人怎么创建
- Bot ID 和 Secret 从哪里拿
- 本地部署和云端部署到底怎么配
- 为什么渠道已经加好了，机器人还是不回复

这篇文章不讲空话，直接带你把整条链路跑通：

**创建企微机器人 -> 绑定 OpenClaw -> 完成 Pairing -> 在企业微信里正常对话**

如果你想要的不是“本地玩玩 AI”，而是一个真正能进入团队协作入口的智能助手，这篇可以直接照着做。

---

## 先说结果：这次接入意味着什么

企业微信一旦接上 OpenClaw，价值不只是“可以聊天”。

更重要的是，你终于有了一个适合企业内部使用的 AI 入口：

- 消息入口在企业微信，团队成员不需要额外换工具
- OpenClaw 负责模型调用、消息处理和后续扩展
- 支持长连接模式，既能回复消息，也支持主动发消息
- 基础对话跑通后，还能继续接智能表格、企微 API、知识库等能力

换句话说，这不是把一个 AI 工具塞进企业微信，而是把企业微信变成了 OpenClaw 的工作台入口。

---

## 开始前，先确认 3 件事

正式操作前，先检查下面三项：

| 条件 | 说明 |
|------|------|
| 企业微信客户端已更新到最新版 | 官方流程基于新版界面 |
| OpenClaw 已经安装并能正常运行 | 本地可用，或已部署在腾讯云 Lighthouse |
| 你有创建企业微信智能机器人的权限 | 后续需要保存机器人配置 |

如果这三项都没问题，就可以直接开始。

---

## 第一步：在企业微信里创建智能机器人

这一部分最关键的地方只有一句话：

**一定要选 API 模式，并且要用“长连接”方式创建。**

很多人后面接不上，问题往往不是出在 OpenClaw，而是第一步就选错了机器人模式。

### 1. 进入企业微信工作台，找到“智能机器人”

![进入智能机器人](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/01.png)

### 2. 创建机器人，选择 API 模式

![创建机器人](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/02.png)

### 3. 选择“长连接”方式，拿到 Bot ID 和 Secret

这两个参数后面绑定 OpenClaw 时会直接用到，建议先复制保存。

![获取 Bot ID 和 Secret](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/03.png)

到这里，企业微信这一侧的准备工作就完成了。

---

## 第二步：把企业微信机器人接到 OpenClaw

这一步分两种情况：

- 你是腾讯云 Lighthouse 部署用户
- 你是本地终端部署用户

两种方式选一种即可。

---

## 方案一：你把 OpenClaw 部署在腾讯云 Lighthouse

如果你已经在腾讯云轻量应用服务器里部署了 OpenClaw，这套流程会比较省事，基本以可视化配置为主。

### 1. 打开 Lighthouse 实例，进入“应用管理”

![进入 Lighthouse 应用管理](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/04.png)

### 2. 在通道里选择“企微机器人（长链接）”

然后把刚才拿到的 **Bot ID** 和 **Secret** 填进去。

![填写 Bot ID 和 Secret](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/05.png)

### 3. 点击“添加并应用”，等待配置生效并重启

![添加并应用](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/06.png)

### 4. 回到企业微信，保存机器人并发消息测试

只要机器人已经能正常回复，就说明接入成功。

![企业微信中测试对话](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/07.png)

如果你本来就是云端部署，建议优先走这条路径，步骤更短，也更适合团队环境。

---

## 方案二：你把 OpenClaw 跑在本地终端

如果你是本机安装 OpenClaw，这部分按命令行一步一步来就可以。

### 1. 安装企业微信插件

```bash
openclaw plugins install @wecom/wecom-openclaw-plugin
```

安装成功后，终端会出现提示信息：

![安装插件成功](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/08.png)

### 2. 启动或重启 OpenClaw 网关

```bash
openclaw gateway start
```

如果你的网关已经在运行，建议重启后再继续配置，避免中间状态影响接入。

### 3. 添加企业微信渠道

```bash
openclaw channels add
```

在 `select channel` 这一步，选择 **企业微信**：

![选择企业微信渠道](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/09.png)

### 4. 输入 Bot ID 和 Secret

![输入 Bot ID 和 Secret](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/10.png)

### 5. 选择 `finish`

![选择 finish](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/11.png)

### 6. 配对方式选择 `Pairing`

这里不要跳过。  
对本地部署来说，`Pairing` 这一步才是真正把当前 OpenClaw 实例和企业微信机器人绑定起来。

![选择 Pairing](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/12.png)

### 7. 完成剩余配置，确认渠道添加成功

![渠道配置成功](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/13.png)

### 8. 回到企业微信，保存机器人并先发一条消息

这时机器人会回一段配对信息，其中会包含一个配置密钥。

![收到配置密钥](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/14.png)

### 9. 复制消息最后一行，粘贴回终端完成配对

这里要注意，终端里真正要用的是消息里的最后一行密钥，不是整段文字全部复制。

![粘贴密钥完成配对](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/15.png)

### 10. 再发一条消息，验证机器人是否已经正常回复

![配对完成后正常对话](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/16.png)

到这里，本地方案也算正式跑通了。

## 这 3 个地方，最容易翻车

### 1. 第一阶段选错了机器人模式

如果不是 API 模式，或者不是长连接方式，后面会非常容易对不上。

### 2. 渠道加上了，但没有完成 Pairing

这是最常见的坑之一。  
本地部署时，只是执行了 `openclaw channels add` 并不等于机器人已经能工作，真正生效还要靠配对完成。

### 3. 配对密钥复制错了

官方文档这块写得比较短，实际操作时很容易误复制。记住一点就行：**复制机器人消息最后一行的密钥内容。**

---

## 基础对话跑通之后，还能怎么扩展

如果你只是想把机器人接起来，到前面其实已经够了。  
但如果你想把它真正做成企业工作流的一部分，后面还有两条非常实用的扩展路线。

### 1. 接智能表格 Webhook

当你在智能表格里开启“接收外部数据”后，系统会生成一个唯一的 Webhook 地址。  
这时候 OpenClaw 就可以继续把消息结果写入智能表格，或者更新已有数据。

![智能表格 Webhook 场景](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/17.png)

这类能力很适合这些场景：

- 销售线索自动入表
- 工单和审批信息同步
- 对话结果自动沉淀
- 结构化信息归档

### 2. 继续调用企业微信 API

如果你希望机器人不只是回答问题，而是进一步联动企业微信应用能力，那么可以继续补齐企业 ID 和应用 Secret。

先在管理后台获取企业 ID：

![获取企业 ID](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/18.png)

再到自建应用里获取应用 Secret：

![获取应用 Secret](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/19.png)

把这些参数发送给已经接好的 OpenClaw 机器人后，就可以进一步获取 `access token`，继续调用企业微信文档等 API。

![进一步调用企微 API](https://raw.githubusercontent.com/liuy-byte/weixin-mp/main/images/wecom-openclaw-official/20.png)

如果你后面打算做文档处理、表格写入、内部流程联动，这一步会很关键。

## 最后说一句

这次企业微信把 OpenClaw 接入流程正式写进帮助中心，真正有价值的地方，不在于“多了一个教程”，而在于企业微信场景终于有了一条标准化、可复用、可扩展的 OpenClaw 接入路径。

如果你只想先跑通最小闭环，记住下面 4 个动作就够了：

1. 在企业微信里创建 API 模式机器人
2. 选择长连接，拿到 Bot ID 和 Secret
3. 在 OpenClaw 里添加企业微信渠道
4. 完成 Pairing，并发消息验证

跑通这一步之后，企业微信就是入口，OpenClaw 就是能力中枢。

后面无论是接智能表格、接知识库，还是接企业内部 API，都会顺很多。
