# CLAUDE.md

## 项目概述

微信公众号**赛博技术派**的文章仓库，内容以 AI 编程工具教程为主，面向国内开发者。

本仓库使用两个 GitHub 仓库管理：

- **weixin-mp-images**（public）：图床，只存图片
- **weixin-mp-articles**（private）：文章，存 md 文件

## 文件命名规范

md 文件以年月日为前缀，格式：`YYYYMMDD-标题.md`

## Git 分支与远程仓库

本地两个分支，分别推送到不同的远程仓库：

- **`main` 分支** → `origin`（weixin-mp-images）：只提交图片
- **`local` 分支** → `private`（weixin-mp-articles）：跟踪所有 md 文件

```bash
# 日常写作（在 local 分支）
git add *.md
git commit -m "update: 文章标题"
git push private local:main

# 推图片到远端
git checkout main
git add images/
git commit -m "chore: add [图片说明]"
git push origin main
git checkout local
git merge main
```

## 公众号写作风格规范

编写新文章时严格遵守以下规范。

### 标题规律

- 以**数字**量化价值：「16个命令」「5-10倍」
- 以**动作词**制造张力：「干翻」「告别」
- 包含**具体收益**：「效率翻倍」「提升明显」
- 格式：`[产品/工具]｜[核心价值/卖点]` 或直接点明效果

### 文章结构

```
1. 开头引用块（blockquote）—— 1-2句描述读者痛点，不超过3行
2. 效率对比表格（可选）—— 传统方式 vs 新方式，量化收益
3. 正文（汉字编号：一、二、三...）
   - 每个功能/概念独立成节
   - 核心能力用 ✅ 列表
   - 实战案例用引用块（模板见下方）
4. 结语 —— 1-2句，直接点明价值
5. 相关链接 —— 代码块包裹 URL，微信内长按复制
```

**实战案例模板**（必须包含：具体场景 + 执行过程 + 量化效果）：

```markdown
> **实战案例**：[具体场景]，[工具/方案]完成了[具体操作]，[量化结果]。
```

### 语言风格

- **短句为主**，节奏感强，避免长难句
- **量化收益**：效率数据必须具体（「提升3.5倍」而非「大幅提升」），有实战支撑
- **口语化但专业**：技术术语保留英文，解释用中文
- 使用 emoji 辅助排版（✅ ❌ ⚡ 💡 🔥），不过度堆砌
- 步骤说明用简洁陈述句，不逐条拆解

### 格式规范

- 主要章节之间用 `---` 分隔
- 对比类内容优先使用表格
- 代码块内配简短中文注释
- 图片统一放 `images/` 目录
- 排版参考：[示例.md](./示例.md) — 包含完整的 Markdown 格式示范

### 禁忌清单

- 模板化的"本文价值"列表（✅ 真实环境实操 / ✅ 完整命令行输出...）
- "担心...？想...？"引导句式
- 写了表格又在正文重复说明
- 章节标题加"解读"、"说明"等冗余词
- "这是**唯一需要提前准备**的内容"等强调废话
- 「如有疑问欢迎留言」等无效客套
- 「总的来说」「综上所述」等套话过渡
- 效率数据无实战支撑随意捏造
- 优化目标：比 AI 初稿减少约 30% 文字量

### 文章末尾模板

```markdown
---

## 相关链接

微信内长按复制链接到浏览器打开。

**[分类名]**
​```
链接名：https://example.com/
​```
```

**注意**：文末不加 GitHub 仓库链接，不加版权声明 blockquote。
